# 新Preview页面逻辑

#功能设计
1. 当用户在chat界面成功设计角色并且不缺内容能够进入preview页面后 **直接更改角色卡状态为已完成**
2. 当角色卡内容为已完成时，仍然展示底部栏，但是由“继续创作“和”完成创建”，改为“继续设计”和”导出作品”

当用户点击继续设计：
    - 修改角色卡状态为未完成
    - 重新返回chat页面

当用户点击导出作品的时候：
    - 跳出弹窗显示“功能开发中”

# 注意事项
- 修改过程中需要考虑工程修改足够全面，不要留下冗余代码
- 底部“继续设计”和”导出作品”的悬浮栏改为使用类似于home页面底部的tabbar的设计
