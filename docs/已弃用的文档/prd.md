**O亿C 项目需求文档**


1. 这个小程序是干嘛的

产品名称：
核心定义：基于小程序平台和 AI Agent 的原创角色（Original Character）创作灵感助手 。
核心痛点：解决 ACGN 爱好者在构思角色时思维零散、逻辑简单、生图争议等问题，帮助用户总结灵感并生成专业角色信息卡 。

2. 主要功能有哪些
2.1 全局导航
底部标签栏 (Tabbar)：
    首页：核心功能区，负责角色卡的展示、管理与新建入口 。
    我的：个人中心，包含用户信息、作品管理及系统设置 。

2.2 首页：角色卡中心
功能描述：
    角色展示：以横向滑动列表形式展示用户已创建的角色卡片，角色卡片为竖向圆角长方形
    状态标识：清晰标注“已完成”或“未完成”的角色状态。（未完成的角色以磨砂效果遮住文字并在中心配上“未完成”的红字，已完成角色展示部分文字信息）
    新建入口：最终中间的一张角色卡为新建角色的入口，是白底配上显眼的“+”按钮，点击跳转至创建流程。

2.3 创建流程：对话交互
对话页面：
    交互形式：仿AI聊天界面设计，用户输入零散想法，Agent 实时回复引导 。
    素材导入：支持用户上传参考图或文字素材给 Agent。
    对话框右上方悬浮有一个圆形按钮用于确认生成
    *特别说明*： 给我一个暴露出来Agent接口，我要接入外面的Coze的API。

生成确认：
    模板选择：提供多种角色信息表模板供用户选择。（先只提供单个模板）
    设计规范：代码设计一个接口用于提供对外确认生成的请求
    结果预览：悬浮窗口展示 Agent 最终汇总的角色设定，确认无误后点击“完成创建” ，或者选择“继续创作”回到对话页面


2.4 我的：用户信息中心
用户档案：支持修改用户名、更换头像等基本个人信息修改 。
我的作品：集中管理所有已生成的 OC 作品 。为一个横向可滑动的列表
系统支持：包含使用指南、意见反馈以及余额充值入口（用于 v2.0 支付闭环准备） 。

3. 界面设计规范 (基于 TDesign)
统一风格： 必须使用 **TDesign Weixin** 这个现成的界面工具包来做，让小程序的按钮、列表、输入框等看起来风格统一、简洁、好看。
交互动效：用起来要流畅，滑动列表、切换页面不能卡。点了按钮、等图片生成、加载更多图片时，都要有提示，让用户知道现在是什么状态。小程序打开速度要快看图片(尤其首页的很多图)加载要快。
规范参考：参考:https://tdesign.tencent.com/miniprogram/getting-started 
安全要求：微信登录过程要安全，不能泄露用户信息。
