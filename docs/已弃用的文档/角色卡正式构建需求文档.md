# 需求
1. 完成数据库的结构设计和云函数的设计
2. 完成下述所有功能，实现完整的角色设计闭环
3. 告诉我需要在微信小程序设计工具中添加哪些更新哪些东西

# 更新功能

## 用户在home页面可以浏览到：   
    1. 加号卡片（添加角色）
    2. 未完成的角色卡（如果有）
    3. 已完成的角色卡（如果有）

## 用户在profile页面可以浏览到：
    1. 在‘我的作品’下的已有角色卡（如果用户账户中有角色卡）
    2. 如果没有角色卡，则添加一行字提示“还没有角色卡，快去创建吧~”

## 用户点击创建新的角色卡：
    弹窗提示是否要创建角色卡

## 用户点击未完成的角色卡：
    1. 进入对话页面，还原原有的对话记录

## 用户在chat页面点击确认创建角色卡：
    向Dify端发送一句Give_Result，得到标准的角色卡内容（参考character_card_design.md）,然后进入preview页面渲染出角色卡。

## 如果没有参考图片上传
    如果没有参考图片上传，用占位图片替代，后续告诉我我该把占位图片放在哪个目录

## 用户在preview页面点击完成创建:
    标记角色卡为已完成，返回主页，弹窗提示角色卡创建已完成。
